Common diagnostic trouble codes (DTCs) explained

June 23, 2026 · 7 min read · Diagnostics

Diagnostic Trouble Codes (DTCs) are the language your vehicle's computer uses to tell you something is wrong. Learning to interpret the most common codes will make you a faster, more confident diagnostician.

Understanding DTC structure

Every OBD2 code follows a 5-character format: PXXXX, where the first letter indicates the system:

  • P: Powertrain (engine, transmission, emissions)
  • B: Body (lights, doors, climate control)
  • C: Chassis (ABS, steering, suspension)
  • U: Undefined/Network (communication faults)

Top 10 codes you'll see regularly

P0101 – Mass or Volume Air Flow (MAF) Sensor Range/Performance

Causes: Dirty MAF sensor (most common), intake leak, vacuum leak, bad sensor

  • Quick fix: Clean MAF with specialized solvent, don't touch the sensor element
  • Diagnosis: Check live MAF data, compare to known-good baseline

P0300 – Random/Multiple Cylinder Misfire Detected

Causes: Spark plugs, coil packs, fuel injectors, fuel pressure, compression issues

  • Key step: Check for cylinder-specific misfire codes (P0301-P0308) first
  • Live data: Monitor fuel pressure, spark timing, and injector pulse width

P0401 – EGR Flow Insufficient

Causes: Clogged EGR valve, bad EGR solenoid, intake carbon buildup

  • Common fix: Carbon cleaning or EGR valve replacement
  • Note: This code is less common on newer vehicles with advanced emission control

P0430 – Catalyst System Efficiency Below Threshold (Bank 2)

Causes: Failing catalytic converter, oxygen sensor issue, engine running rich/lean

  • Check first: Oxygen sensor voltages (upstream vs. downstream)
  • Reality check: Catalytic converters fail more often on higher-mileage vehicles
💡 Tip: Always address the root cause, not just the code. A P0430 code might be an oxygen sensor, not a converter.

P0505 – Idle Air Control System Malfunction

Causes: Vacuum leak, dirty throttle body, bad idle air control valve, intake leak

  • diagnosis: Check idle RPM stability with scope or graphing tool
  • Quick win: Throttle body cleaning solves this more often than you'd think

P0600-P0699 – Communication faults

Causes: CAN bus issues, corroded connectors, module failures, wiring problems

  • Investigation: Check DLC pins 6 and 14 for CAN bus integrity
  • Scope work: Monitor CAN traffic during fault trigger

C1201 – ABS fault detected

Causes: wheel speed sensor failure, ABS pump/motor issue, wiring problems

  • Start here: Check all four wheel speed sensors with a scope
  • Signal quality: Low signal voltage or noise can trigger false faults

P0700 – Transmission Control System Malfunction

Causes: Transmission fault (get transmission-specific codes), wiring issue

  • Important: This is usually a gateway code; the real issue is in the supporting DTCs
  • Read full list: Always pull all DTCs, not just the first one

U0100 – Lost Communication With ECM/PCM

Causes: CAN bus down, ECM power issue, wiring or connector fault

  • Check: Power and ground to the ECM using a multimeter
  • Next: Verify CAN lines are responding with a scope

Diagnostic workflow

When you see a code:

  • 1. Understand the code: What system, what condition triggered it?
  • 2. Check for pending DTCs: Sometimes earlier faults cause secondary codes
  • 3. Use live data: Monitor relevant parameters while recreating the issue
  • 4. Verify with waveforms: A digital multimeter shows voltage; a scope shows the story
  • 5. Confirm the fix: Clear the code, road test, verify it doesn't return
🔧 Pro tip: Keep a reference chart of the top 20 codes in your bay. Faster diagnosis = happier customers = more time for the next job.
