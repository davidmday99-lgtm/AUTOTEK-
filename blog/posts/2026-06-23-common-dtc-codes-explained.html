Common DTC Codes Explained

June 23, 2026 · 7 min read · Diagnostics

Diagnostic Trouble Codes (DTCs) can be mysterious at first, but understanding their structure and the most common codes you'll encounter will accelerate your diagnostic process and help you communicate accurately with customers.

Understanding DTC Structure

Every OBD2 code follows a five-character format: one letter followed by four numbers.

  • First character (letter): System type (P=Powertrain, B=Body, C=Chassis, U=Network)
  • Second character (number): Standard (0) or Manufacturer-specific (1)
  • Third character (number): System (fuel/air, ignition, emissions, etc.)
  • Fourth & fifth (numbers): Specific fault
💡 Example: Code P0300 breaks down as: Powertrain (P), Standard (0), Ignition/Fuel (3), Random cylinder misfire detected (00).

Powertrain Codes (P0xxx)

The most common codes you'll see relate to engine operation:

  • P0300: Random/multiple cylinder misfire—check spark plugs, fuel injectors, compression
  • P0101: Mass airflow (MAF) sensor out of range—usually needs cleaning or replacement
  • P0128: Coolant thermostat malfunction—often points to stuck thermostat or sensor fault
  • P0420: Catalyst system efficiency below threshold—O2 sensor or converter issues
  • P0500: Vehicle speed sensor malfunction—affects transmission shift quality and cruise control

Oxygen Sensor Codes

O2 sensor problems are extremely common and usually straightforward to diagnose:

  • P0130-P0135: O2 sensor circuit issues (upstream)
  • P0136-P0161: O2 sensor circuit issues (downstream)
  • Diagnosis tip: Look at live data fuel trim—bank 1 short/long trim values tell you if the sensor is actually working or just reporting a fault

Transmission Codes (P0700+)

Transmission DTCs often appear alongside engine codes:

  • P0700: Transmission control system malfunction—umbrella code, dig deeper into sub-codes
  • P0741: Torque converter clutch circuit—usually solenoid or fluid issue
  • P0750: Shift solenoid A malfunction—can cause hard shifts or limp mode

The Difference Between Code Severity

Not all DTCs are created equal. Pay attention to severity:

  • Type A (MIL on immediately): Critical emissions or safety issue
  • Type B (conditional): Fault condition must repeat multiple times before MIL lights
  • Type C (informational): Doesn't trigger check engine light but still logged
🔧 Pro tip: A flashing check engine light (Type A) means stop driving immediately—there's either a severe misfire or a safety issue. This is different from a steady light.

Pending vs. Confirmed vs. Historical

Understanding code status helps with diagnostics:

  • Pending: Fault detected once but not confirmed—drive vehicle to determine if it repeats
  • Confirmed: Fault detected multiple times—this is the actual problem
  • Historical: Previously detected but hasn't repeated—may be intermittent

Why Not to Ignore Pending Codes

Many techs clear pending codes, but this can hide intermittent issues:

  • Pending codes often develop into confirmed codes within 100-500 miles
  • Intermittent issues are harder to diagnose, but pending codes give you a head start
  • Document pending codes and have the customer monitor—they'll help you catch real problems early
